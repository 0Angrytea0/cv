# Лабораторная работа 1 (вариант на 4) — семантическая сегментация OpenSprayerSeg

## Краткое описание

В работе выполнено исследование моделей семантической сегментации на датасете OpenSprayerSeg.  
Задача состоит в выделении пикселей сорняков на изображениях поля. Такая постановка имеет практический смысл для систем точечного опрыскивания, где гербицид вносится только по зонам сорняков, а не по всему полю.

Датасет: https://www.kaggle.com/datasets/thatawkwardguy/opensprayerseg


## Бизнес-задача

**Практическая задача:** автоматическое выделение сорняков на изображениях поля для систем точного земледелия.

**Зачем это нужно:**  
если модель умеет находить сорняки на уровне пикселей, можно выполнять более точечное внесение гербицидов, снижать расход химикатов и уменьшать воздействие на культурные растения.

## Выбор начальных условий

### Выбранный датасет

Для работы выбран датасет OpenSprayerSeg. Он подходит для реальной прикладной задачи сегментации сорняков на полевых изображениях.

В ноутбуке дополнительно была выполнена проверка разметки. Выяснилось, что цвета в `class_dict.csv` не совпадают с фактическими цветами масок, поэтому для корректной подготовки данных был реализован безопасный декодер бинарных масок.

### Выбранные метрики

Для оценки качества использовались:
- Dice coefficient — основная метрика сравнения
- IoU
- Precision
- Recall
- Pixel Accuracy

Для бинарной сегментации сорняков Dice хорошо отражает качество перекрытия предсказанной и истинной маски.


## Бейзлайн и оценка качества

В качестве библиотечных baseline-моделей из `segmentation_models.pytorch` были выбраны:
- Unet + ResNet18 — сверточная модель
- Segformer + MiT-B0 — трансформерная модель

### Результаты baseline

| Модель | Test Dice | Test IoU | Precision | Recall |
|---|---:|---:|---:|---:|
| Unet_resnet18 | 0.9045 | 0.8256 | 0.9050 | 0.9040 |
| Segformer_mit_b0 | 0.8925 | 0.8059 | 0.9310 | 0.8571 |

### Вывод по baseline

На этапе baseline лучшей моделью по метрике Dice стала Unet_resnet18.

## УУлучшение бейзлайна

### Проверяемые гипотезы

В ноутбуке были проверены следующие гипотезы:
1. базовый пайплайн без улучшений
2. добавление аугментаций
3. аугментации + комбинированный loss BCE + Dice
4. итоговый improved baseline: аугментации + BCE+Dice + `weight_decay` + scheduler + early stopping

### Что вошло в improved baseline

В улучшённый baseline были включены:
- аугментации `HorizontalFlip`, `VerticalFlip`, `RandomRotate90`, `ShiftScaleRotate`, `RandomBrightnessContrast`, `GaussNoise`
- комбинированный BCE + Dice loss
- `weight_decay`
- scheduler `ReduceLROnPlateau`
- early stopping

### Результаты improved baseline

| Модель | Test Dice | Test IoU | Precision | Recall |
|---|---:|---:|---:|---:|
| Unet_resnet18 | 0.9000 | 0.8183 | 0.9285 | 0.8733 |
| Segformer_mit_b0 | 0.8972 | 0.8136 | 0.9402 | 0.8580 |

### Сравнение с baseline

Для Unet_resnet18 улучшенный baseline не дал прироста

Для Segformer_mit_b0 наблюдается небольшой рост относительно baseline

Лучшая библиотечная модель по итогам всех экспериментов осталась Unet_resnet18 baseline, если ориентироваться на фактический `test_dice` из итоговой таблицы ноутбука
